================================================================================
                        DEPENDENCY RELATIONSHIPS
================================================================================

PCI Bus Driver
  ├─► ACPI Driver (requires PCI for device info)
  ├─► Storage Drivers (PCI devices)
  ├─► Display Driver (PCI GPU)
  ├─► USB Controller (PCI device)
  └─► Network Driver (PCI NIC)

ACPI Driver
  └─► IRQ Controller (ACPI configures interrupts)

IRQ Controller
  ├─► Timer (needs IRQ 0)
  ├─► Input Drivers (PS/2 uses IRQ 1, 12)
  ├─► Storage Drivers (AHCI/NVMe use MSI/MSI-X)
  └─► USB Controller (xHCI uses MSI/MSI-X)

================================================================================
                          LOAD ORDER RULES
================================================================================

1. Infrastructure First (PCI, ACPI, IRQ, Timer)
   - These provide services for all other drivers
   - Must load before any device drivers

2. Storage Drivers Early
   - Needed for loading additional drivers from disk
   - Needed for accessing root filesystem

3. Display/Input After Storage
   - Can load after core infrastructure
   - Not critical for boot, but needed for user interaction

4. USB After Basic I/O
   - USB devices can provide additional input/storage
   - Hot-plug capable, can load late

5. Network Last
   - Not critical for local boot
   - Can be loaded on-demand

================================================================================
                            ERROR HANDLING
================================================================================

Error Scenario: Driver fails to load
────────────────────────────────────

    [Driver Init Attempt]
              │
              ├─► Success? ──► Continue boot
              │
              └─► Failure ──► Log error
                                   │
                                   ├─► Critical driver? ──► Panic
                                   │
                                   └─► Optional driver ──► Continue
                                                           (Graceful degradation)

Error Scenario: Device not found
─────────────────────────────────

    [PCI Scan]
         │
         ├─► Device found? ──► Load driver
         │
         └─► Not found ──► Skip driver
                                 │
                                 └─► System continues

Error Scenario: Dependency not met
───────────────────────────────────

    [Driver Load Request]
              │
              ├─► Dependencies satisfied? ──► Load driver
              │
              └─► Not satisfied ──► Defer loading
                                         │
                                         └─► Retry after dependency loads
