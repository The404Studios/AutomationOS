# AutomationOS — Roadmap to Done (master plan)
Synthesized from a 100-agent research+design+adversarial-review swarm (2026-06-20): 45 areas designed to fully-working, each stress-reviewed, then 10 track architects produced dependency-ordered execution plans. This is the *best way forward*.
> Proof discipline (non-negotiable): every item lands via verify-vs-HEAD -> build_all -> 43/43 smoke (+ targeted runtime test where the smoke doesn't exercise it) -> atomic commit. Hardware-gated items need the physical T410.

## Area keystones (highest-leverage per area)
- **sched-coop** (kernel-sched): Fix scheduler_yield_requeue's unlock-relock race (RACE-001) by re-checking on_queue after lock reacquire or by moving process_ref into the single critical section. This is the only finding that blocks SMP correctness; if two CPUs call yield_requeue on the same task, the linked list gets corrupted. Proof: recompile + run 43-smoke (cooperative path only on uniprocessor).
- **sched-preempt** (kernel-sched): Priority-aware time_slice reset in all dispatch paths (CRETURN + IRETQ first-dispatch) so processes receive fair CPU quantums matching their nice level instead of stale or uninitialized slice values.
- **smp** (kernel-sched): Make SMP_SCHED_DISPATCH the default scheduler mode when SMP_FOUNDATION is enabled, with per-CPU runqueues + signal dispatch + preempt_count init properly wired. This is the single highest-leverage change: it forces all infrastructure to be used in CI/boot path, surfaces integration bugs immediately, and unlocks multi-CPU execution of actual workloads (not just AP co-processor mode).
- **futex** (kernel-sched): Fix futex_get_phys_addr() to use PHYS_TO_DIRECT(phys) instead of bare casts—this is the single blocker preventing the kernel from handling process identity-map splits (exec, CoW) without crashing. Once that lands, the timeout stub becomes next leverage point (FUTEX_WAIT_TIMEOUT + wake_deadline wiring).
- **mem-vmm** (kernel-mem): Fix CR3 masking to use 0x000FFFFFFFFFF000ULL consistently (not & ~0xFFF) at line 136 and 182 of vmm.c + line 182 in user_range_is_accessible. This prevents non-canonical pointers when CR3_NO_FLUSH (bit 63) is set with PCID enabled, which #GPs on subsequent table derefs. Blocks multiple downstream fixes.
- **mem-cow-fork** (kernel-mem): Inherit parent signal handlers (sig_handlers[32] + sig_mask) into child at sys_fork line 330-520, mirroring thread_create's pattern at process.c:476-477. This unblocks POSIX fork compliance and enables signal-aware child programs. Proof: existing fork smoke test + targeted signal inheritance test (fork parent with SIGTERM handler, verify child inherits and can handle it).
- **mem-pmm** (kernel-mem): Fix PHYS_TO_DIRECT violations in pmm_free_page/pmm_free_pages SLAB_MAGIC guards (lines 533, 898 in pmm.c) — these are live under process CR3 where identity map is mutated, causing page-fault or data corruption on shadowed frames. This is the single highest-leverage fix for memory safety.
- **syscall-boundary** (kernel-syscall): Fix sys_map_file's fixed VA vulnerability + implement per-process VA allocation, then harden validate_user_buffer to actually test page accessibility (both are blocking for safe initrd-backed file mapping to userspace).
- **signals** (kernel-syscall): 
**Fix g_sig_frame per-CPU array + handler restorer validation**: The per-CPU architecture is the high-leverage change that unlocks SMP safety. Once the frame is per-CPU (6 files: kill.c, syscall.asm, update references), add mandatory sig_restorer validation at sigaction time so handlers cannot silently fail. Together these fix 2 P1s (SMP crash risk + handler delivery assumption).

- **poll-epoll** (kernel-syscall): Fix the timeout unit mismatch in all three syscalls (poll/select/epoll_wait) by adding #include "time.h" and using ms_to_ticks() — this is the highest-leverage fix that unblocks realistic testing and use.
- **security-cap** (kernel-syscall): Uncomment and wire seccomp_check() into the syscall dispatcher at the correct placement (after fast-path checks, before handler execution) and ensure seccomp_init()/capability_init() are called from kernel_main. This single keystone unblocks end-to-end seccomp filtering and allows testing. Without it, the entire subsystem is dead code regardless of implementation quality.
- **net-l3** (net): Implement net_icmp_port_unreachable() + refactor udp_input() to receive full IP packet context. This unblocks Port Unreachable error reporting (P1 c=0.98) and creates the plumbing for ICMP Time Exceeded (P2 c=0.95). The latter requires saving orig_ip_hdr in frag_reasm_t—both are a 40-line patch once port_unreachable lands.
- **net-tcp** (net): Fix cwnd accounting (item 1): remove blind `in_flight = 0` reset on rt_pending clear; only decrement on confirmed snd_una advance. This is the most immediate blocker to proper flow control and enables fixing the others.
- **net-udp-dns** (net): Fix DNS source validation (P1, 5-line surgical fix) + implement UDP port unreachable ICMP error response (P1, requires L3 refactor to pass full IP packet context through UDP demux layer, not a standalone fix). These two unblock ICMP error path and prevent silent spoofing attacks. Together they shift UDP/DNS from "works locally" to "production-ready error handling."
- **net-sock** (net): Implement a SOFTIRQ_RX handler + wiring to NIC IRQs (if available) and syscall hooks to deliver pending network data on blocking poll/recv. This is the leverage point: it unblocks both interrupt-driven RX (a fundamental OS feature) and allows socket-blocking semantics (select/poll returning on network events rather than spinning). The current poll-mode is a deliberate simplification, not a missing feature. The real gaps are the guard-rail bugs in socket management (collisions, state matching) and silent failures in UDP that should emit errors.
- **nic-e1000** (net): Integrate interrupt-driven RX with ISR + EOP validation: register real IRQ handler that calls e1000_rx_poll() on device interrupt, validate RXD_STAT_EOP in descriptor inspection to handle multi-descriptor frames correctly. This unblocks threshold-driven/event-driven RX instead of busy polling, critical for preemptive scheduler + production workloads.
- **wifi-iwl** (wifi-hw): Implement REPLY_RXON association command in iwl-ops.c (fill channel/BSSID/rates from target BSS, submit via iwl_send_cmd, wait for response). This unlocks the whole control-plane path: once REPLY_RXON succeeds, REPLY_RXON_TIMING + REPLY_ADD_STA complete the association, and the firmware will respond to REPLY_SCAN_CMD (already written). The scan brick is complete; association is the hardware gating factor.
- **wifi-sim** (wifi-hw): Track connected AP BSSID through entire stack (sim backend → status syscall → userspace) — once that's wired, the remaining fixes (CSPRNG scalars, GTK, KAT vectors) follow as refinements. The BSSID flow is the critical integration spine: without it, applications cannot verify which AP they joined.
- **t410-hw-program** (wifi-hw): REPLY_RXON + REPLY_ADD_STA implementation: Build the control-plane association and station commands (struct iwl_rxon_cmd and struct iwl_add_sta_cmd) with correct family-specific layouts, serialize them with real BSS/channel/BSSID/rates, submit via iwl_send_cmd(), and integrate the responses into the state machine. This unblocks connect→authenticate→associate→4-way handshake flow and is the load-bearing pivot for hardware validation.
- **vfs** (storage-fs): Wire missing SYS_LSEEK syscall + fix page_cache_read/write corruption: these unblock (1) proper file seeking for all userspace, (2) re-enabling the unified page cache for 10-100x speedup, and (3) proper ramfs+ext2 integration. Together they restore the VFS vision from workaround state to seamless multi-filesystem architecture.
- **ramfs** (storage-fs): Fix the page cache data corruption (P0). The infrastructure is there (page_cache.c:40–67, cache_flush_entry, LRU eviction), but reads/writes bypass it with explicit comments citing "write-then-read content mismatch." Once cache correctness is verified via targeted unit tests (page-aligned offset math, dirty-page flushing, partial-page boundary handling), re-enable page_cache_write in ramfs_write and remove the direct-memcpy bypass. This unlocks all other improvements (durability via flushed cache, CoW optimization, sparse-gap correctness).
- **ext2** (storage-fs): Wire ext2_file_ops into vfs_open() by checking inode->ops or sb->type. This is load-bearing: without it, ext2 reads never reach ext2_vfs_read, all reads fail. One-line fix opens the door to all subsequent write work.
- **fat32** (storage-fs): Fix fat32_mount block device lookup (use block_get_device(source)) and wire file ops correctly in vfs_open by checking inode->ops and assigning corresponding file_ops. These two changes unblock the entire read path.
- **diskfs** (storage-fs): Wire diskfs.c into the kernel build (Makefile), fix the block-count validation bug (line 468: DFS_BITMAP_BITS -> DFS_BLOCK_COUNT), and add a syscall entry point (SYS_DISKFS_READ/WRITE/CREATE) so userspace can actually use persistent storage. Once compiled + syscalls wired, the existing file API (diskfs_read/write/create/open/unlink) will be accessible and proven via smoke_persist.sh.
- **block-ahci** (storage-fs): Fix sector size mismatch between syscall layer (hardcoded 512) and driver (variable, up to 4096): query actual sector_size from port before computing buffer len in sys_blk_read/write. This is the highest-impact fix (P1, c=0.88) that unblocks 4K-sector drives and prevents buffer overrun/memory corruption.
- **nvme** (storage-fs): Add nvme_init() call to kernel.c boot sequence (5 lines) — unblocks all downstream work. Single highest-leverage change: without this, the entire driver stays dormant regardless of other fixes.
- **audio-hda** (drivers-io): Remove HDA_ENABLE compile gate and implement runtime platform detection to safely enable HDA on QEMU while gatekeeping T410 via boot-phase safe-fail logic. This unblocks all builds and eliminates the primary reason audio was never called in production boots.
- **fb** (drivers-io): Add SFENCE after WC MTRR setup (P1 correctness), then implement AP-side WC setup with BSP-passed globals (P2 performance). These two fixes ensure WC is both correct and works on all CPUs. Readback validation (P2) adds observability. Everything else is incremental (PAT fallback, physical address width limits).
- **input-ps2** (drivers-io): Add spinlock to evdev_device_t and synchronize all client list mutations (open/close) and ring buffer accesses (event push/read) with spin_lock_irqsave/spin_unlock_irqrestore. This eliminates the two P1 concurrency races and unblocks real multi-core and high-frequency input testing.
- **usb** (drivers-io): Fix the HID interrupt TD resubmission chain in uhci_poll() and properly link interrupt QHs in the frame list (linked chain instead of replacement) so that multiple HID devices can coexist and continuous polling works. This unblocks continuous keyboard/mouse input and multi-device support without major architectural changes.
- **pci** (drivers-io): Add pci_find_class_by_index(class, subclass, prog_if, index) and pci_find_device_by_index(vendor, device, index) APIs to iterate multiple devices. This unblocks multi-device support in NVMe, Ethernet, and other drivers while requiring zero changes to existing caller patterns.
- **cc-front** (toolchain-ide): Implement minimal preprocessor (#include search + macro expansion). This unblocks real programs — currently even `#include <stdio.h>` is silently stripped, making 95% of real C code unparseable.
- **cc-back** (toolchain-ide): Implement do-while codegen in cc_codegen.c (5 lines, high leverage) — unblocks loop-construct parity; second implement goto/label two-pass collection (20 lines, moderate leverage) — unblocks control flow patterns. Together these add essential statement coverage without architectural changes. Indirect calls require assembler "call rax" support first (15-line encoder extension in as_x64.c), then 30 lines in cc_expr.c to evaluate callee expression.
- **ide** (toolchain-ide): Create userspace/apps/ide/Makefile that mirrors taskmanager (freestanding, wl_client + bitfont deps) and wire 'ide' into userspace/apps/Makefile. This is pure build plumbing (low-risk, non-functional) and unblocks the remaining fixes. Once compiled and runnable, the IDE itself becomes the working environment for validating P1-P2 fixes.
- **browser** (web): Implement external script loading (fetch + execute in document order) — unblocks any multi-file JS project; highest ROI for content compatibility with minimal implementation complexity (HTTP GET + lexical re-execution wrapper).
- **tls-crypto** (web): Fix ECDSA signature verification to support SHA-384/512 at all three layers (TLS ServerKeyExchange, TLS ClientHello cipher suites, x509_verify_chain) — this unlocks modern ECDSA certificates that 30% of servers now use. Highest leverage single fix: accept ECDHE_ECDSA_WITH_AES_256_GCM_SHA384 + pass actual hash length to p256_ecdsa_verify.
- **comp-damage** (platform-ux): Add full-damage cooldown + damage_add_full() call immediately after every anim_begin() for animation phases (PH_SNAPPING/MINIMIZING/RESTORING), and track state changes (focus, hover, drag) via prev-frame comparisons, setting g_full_damage_cooldown when they change. This unifies the damage model: mark all screen mutations as either "direct damage rect" or "full-scene flag" — the compositor will then enforce scissor safety across all code paths.
- **comp-input** (platform-ux): Fix evdev spinlock races (P1 c=0.94): Protect client list mutations and ring-buffer access with spinlock, eliminating TOCTOU bug where evdev_open/evdev_close run without locking while IRQ reads. This unblocks real input delivery and is 2-week effort with high confidence. Then wire evdev blocking read to wait queues (P2 c=0.98), making input truly available to userspace. These two fixes enable the compositor to actually receive keyboard input events reliably.
- **comp-render** (platform-ux): Fix cz_text Y scissor bypass (P1 c=0.95): add Y-bounds parameters to font2_draw_cell_clip calls — this is the single highest-impact issue affecting all window titlebars, panel text, and dock labels. Fixes damage invariant violation and visual correctness across 20+ text render sites.
- **libc-stdio** (platform-ux): Implement SYS_LSEEK syscall (kernel) and wire lseek() in userspace. This unblocks fseek/ftell which in turn unblock file-I/O-based formatted input (scanf from files). lseek is the foundation for all file positioning; it's the single highest-leverage fix because it repairs three dependent systems: ftell (position tracking), fseek (positioning), and formatted input from non-stdin sources.
- **libc-sig-time** (platform-ux): Fix raise() to call kill(getpid(), sig) instead of invoking handlers directly — this routes through the kernel's real signal delivery, respects masks, applies default actions, and fixes the core semantic violation affecting any code using raise(SIGTERM/SIGKILL).
- **agent-rail** (platform-ux): Add O_NOFOLLOW to all file-mutating tool_* opens (tool_write, tool_rm, tool_rollback) to close symlink-attack surface and prevent adversarial redirection of writes to protected system files. This is the highest-leverage fix: blocks a whole class of capability-bypass attacks with single-digit line changes across 3 tools.
- **ai-platform** (platform-ux): Implement the telemetry ring-buffer system with minimal instrumentation (5-6 hot points: kmalloc, process create/exit, context switches, I/O ops) and wire kernel-side SYS_PROC_CTL handlers to prove the control path works end-to-end with a single tuning parameter.
- **ux-refinement** (platform-ux): Fix damage scissor enforcement for animation phases (minimize/restore/snap) by adding g_full_damage_cooldown guards, ensuring visual coherence during window state transitions and preventing ghosting artifacts.
- **release-eng** (platform-ux): Unified build system with IDE included + automated release pipeline (versioning → signing → staged distribution) + reproducible builds validated on T410 hardware
