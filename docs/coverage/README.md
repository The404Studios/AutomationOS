# AutomationOS Code Coverage Reports

This directory contains HTML coverage reports generated by lcov/gcov.

## Quick Start

```bash
# Generate coverage report
make coverage

# Open report in browser
xdg-open docs/coverage/index.html  # Linux
open docs/coverage/index.html      # macOS
start docs/coverage/index.html     # Windows
```

## Coverage Metrics

### Line Coverage
- **Green lines:** Executed during testing
- **Red lines:** Not executed (missing coverage)
- **Yellow lines:** Partially executed (branch not fully covered)

### Branch Coverage
- Shows which branches (if/else, switch) were taken
- Critical for testing error paths

### Function Coverage
- Shows which functions were called
- Helps identify dead code

## Understanding Coverage Reports

### Good Coverage
```c
void* allocate(size_t size) {
    if (size == 0) {        // ✓ Both branches tested
        return NULL;        // ✓ Error path covered
    }
    
    void* ptr = malloc(size); // ✓ Executed
    if (!ptr) {               // ✓ Both branches tested
        return NULL;          // ✓ OOM path covered
    }
    
    return ptr;               // ✓ Success path covered
}
```
**Coverage:** 100% line, 100% branch

### Poor Coverage
```c
void* allocate(size_t size) {
    if (size == 0) {        // ✗ Only true branch tested
        return NULL;
    }
    
    void* ptr = malloc(size); // ✓ Executed
    if (!ptr) {               // ✗ False branch never tested
        return NULL;          // ✗ OOM path not covered
    }
    
    return ptr;
}
```
**Coverage:** 100% line, 50% branch (missing error paths!)

## Coverage Goals

| Component | Line Coverage | Branch Coverage |
|-----------|---------------|-----------------|
| System Calls | > 90% | > 80% |
| Memory Management | > 85% | > 75% |
| Scheduler | > 80% | > 70% |
| Drivers | > 70% | > 60% |
| Userspace | > 75% | > 65% |
| **Overall** | **> 80%** | **> 70%** |

## How to Improve Coverage

1. **Identify uncovered lines** in the HTML report (red lines)
2. **Write tests** to execute those lines
3. **Focus on error paths** (often missed)
4. **Run coverage again** to verify improvement

Example:
```bash
# 1. Run coverage
make coverage

# 2. Open report, identify red lines in heap.c
xdg-open docs/coverage/kernel/core/mem/heap.c.gcov.html

# 3. Write test to cover those lines
vim tests/unit/test_heap.c

# 4. Run coverage again
make coverage

# 5. Verify improvement
```

## CI Integration

Coverage reports are automatically generated on every commit via GitHub Actions.

- **Pull Requests:** Get coverage comments
- **Main Branch:** Coverage tracked over time
- **Codecov:** Detailed coverage trends at https://codecov.io/

## Troubleshooting

### No coverage data
```bash
# Clean and rebuild
make coverage-clean
make coverage
```

### Coverage report is empty
```bash
# Ensure tests ran successfully
make -C tests/unit run

# Check for .gcda files
find . -name "*.gcda"
```

### lcov not found
```bash
# Install lcov
sudo apt-get install lcov  # Ubuntu/Debian
brew install lcov          # macOS
```
