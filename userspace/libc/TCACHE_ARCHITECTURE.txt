================================================================================
CACHE HIT EXAMPLE (Optimal Case)
================================================================================

ITERATION 1:
    malloc(64):
        1. Check tcache bin 3 → MISS (empty)
        2. Allocate from arena → 1 SYSCALL (if arena grows)
        3. Return ptr0

    free(ptr0):
        1. Check tcache bin 3 → count=0 (not full)
        2. Cache ptr0 → bin[3].entries[0] = ptr0, count=1
        3. Return (0 syscalls!)

ITERATION 2-10000:
    malloc(64):
        1. Check tcache bin 3 → HIT! (ptr0 cached)
        2. Pop from cache → return ptr0 (0 syscalls!)

    free(ptr0):
        1. Check tcache bin 3 → count=0 (we just took it)
        2. Cache ptr0 again → bin[3].entries[0] = ptr0
        3. Return (0 syscalls!)

TOTAL SYSCALLS: 1 (first allocation only)
WITHOUT TCACHE: 20,000 (1 per malloc, 1 per free)
REDUCTION: 20,000x ✅


================================================================================
CACHE SATURATION EXAMPLE
================================================================================

Allocate 20 chunks of 64 bytes:
    ptrs[0..19] = malloc(64) × 20
    All come from arena (no cache yet)

Free all 20:
    free(ptrs[0]):  → tcache bin 3, count=1
    free(ptrs[1]):  → tcache bin 3, count=2
    ...
    free(ptrs[15]): → tcache bin 3, count=16 (FULL!)
    free(ptrs[16]): → OVERFLOW! Goes to arena free-list
    free(ptrs[17]): → arena free-list
    free(ptrs[18]): → arena free-list
    free(ptrs[19]): → arena free-list

Reallocate 20 chunks:
    malloc(64) #1:  → cache hit (pop from bin 3), count=15
    malloc(64) #2:  → cache hit, count=14
    ...
    malloc(64) #16: → cache hit, count=0 (cache exhausted)
    malloc(64) #17: → arena allocation
    malloc(64) #18: → arena allocation
    malloc(64) #19: → arena allocation
    malloc(64) #20: → arena allocation

RESULT:
    First 16 frees:  Cached (fast)
    Last 4 frees:    Arena (slower, but still fast)
    First 16 allocs: From cache (fast)
    Last 4 allocs:   From arena (slower)


================================================================================
MULTI-SIZE PATTERN
================================================================================

Application allocates different sizes:

    void* p16  = malloc(16);   → tcache bin 0, miss, arena alloc
    void* p64  = malloc(64);   → tcache bin 3, miss, arena alloc
    void* p128 = malloc(128);  → tcache bin 7, miss, arena alloc
    void* p256 = malloc(256);  → tcache bin 15, miss, arena alloc

Free all:

    free(p16);   → cached in bin 0
    free(p64);   → cached in bin 3
    free(p128);  → cached in bin 7
    free(p256);  → cached in bin 15

Reallocate (different order):

    void* p64_2  = malloc(64);   → HIT! (from bin 3)
    void* p16_2  = malloc(16);   → HIT! (from bin 0)
    void* p256_2 = malloc(256);  → HIT! (from bin 15)
    void* p128_2 = malloc(128);  → HIT! (from bin 7)

RESULT: 100% cache hit rate! Each size has its own bin.
