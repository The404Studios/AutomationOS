┌────────────────────────────────────────────────────────────────────────────────────────┐
│                              DATA FLOW EXAMPLE                                         │
│                         (User presses 'A' key on keyboard)                             │
└────────────────────────────────────────────────────────────────────────────────────────┘

 1. USB Keyboard:        User presses 'A' key
                         ↓
 2. USB Hardware:        Generates USB report [0x00, 0x00, 0x04, ...]
                         ↓
 3. XHCI Controller:     Interrupt transfer complete, data in memory
                         ↓
 4. USB Core:            Calls interrupt callback
                         ↓
 5. HID Driver:          hid_interrupt_callback(report)
                         ├─> Parse boot protocol report
                         ├─> Compare with previous state
                         ├─> Detect: Key 0x04 newly pressed
                         └─> Translate: 0x04 → KEY_A
                         ↓
 6. Input Subsystem:     input_report_key(dev, KEY_A, PRESSED)
                         ├─> Create event: {timestamp, KEY, KEY_A, 1}
                         ├─> Add to device queue
                         └─> Add to global queue
                         ↓
 7. Application:         event = input_get_event()
                         ├─> Dequeue from global queue
                         └─> Process: Display 'A' character

Time: ~5-10ms (USB polling interval + processing)

KEYBOARD REPORT (8 bytes):
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  Byte 0  │  Byte 1  │  Byte 2  │  Byte 3  │  Byte 4  │  Byte 5  │  Byte 6  │  Byte 7  │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ Modifiers│ Reserved │  Key 1   │  Key 2   │  Key 3   │  Key 4   │  Key 5   │  Key 6   │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
  Bit 0: Left Ctrl      Up to 6 simultaneous key presses
  Bit 1: Left Shift     Values are USB HID usage codes
  Bit 2: Left Alt       0x00 = No key pressed
  Bit 3: Left GUI       0x04 = 'A' key
  Bit 4: Right Ctrl     0x05 = 'B' key
  Bit 5: Right Shift    etc.
  Bit 6: Right Alt
  Bit 7: Right GUI

MOUSE REPORT (3-4 bytes):
┌──────────┬──────────┬──────────┬──────────┐
│  Byte 0  │  Byte 1  │  Byte 2  │  Byte 3  │
├──────────┼──────────┼──────────┼──────────┤
│  Buttons │  Delta X │  Delta Y │  Wheel   │
└──────────┴──────────┴──────────┴──────────┘
  Bit 0: Left button    Signed 8-bit values
  Bit 1: Right button   -127 to +127
  Bit 2: Middle button  Positive = right/down
                        Negative = left/up
